# t-shirt-designer

通用的 t-shirt 设计器, 可以通过传入不同的模型数据和预设部位来适配不同的 t-shirt 模型

## 整体界面

左侧: 渲染区, three.js 接管

右侧: 设计区, 可以给预设的部位修改颜色或添加贴图

## 渲染区

由传入的模型数据渲染 t-shirt 模型, 模型数据包含顶点、纹理坐标、法线等信息

- 使用 three.js 渲染 t-shirt 模型
- 模型可以旋转、缩放、平移
- 模型可以点击, 点击后右侧设计区需滚动到对应部位的设计选项

## 设计区

由传入的预设的部位组成, 每个部位包含颜色选择和贴图上传功能

- 预设部位: 袖子、前片、后片、领口等, 根据传入的预设信息确定, 本身不解析模型
- 颜色选择: 可以选择颜色, 选择后对应部位的颜色会改变
- 贴图上传: 可以上传图片作为贴图, 上传后对应部位会显示贴图, 贴图可以调整位置和缩放、旋转

修改颜色贴图会实时更新渲染区的模型显示, 通过 three.js 的材质系统实现颜色和贴图的叠加效果

## 配置信息

传入的信息:

- 模型数据, 本组件不作任何解析直接丢给 three.js 渲染
- 预设部位信息: 包含部位名称、对应的模型部分标识等, 用于关联设计区的选项和渲染区的模型部分
  - 其他配置项: 如默认颜色、贴图选项等, 可以指定某个部位只能从指定的颜色或贴图选项中选择

## 可以保存的设计

设计状态可以保存为 JSON 对象, 包含每个部位的颜色和贴图信息(记录 URL 或 base64 数据), 以便后续加载或分享

- 希望可以兼容 vue 的 modelValue 模式, 以便在 vue 项目中使用时可以通过 v-model 双向绑定设计状态

## 开发方案

不使用 React 或 Vue 等框架, 直接使用原生 JavaScript 和 HTML/CSS 开发, 以保证组件通用、易集成

组件的核心代码放在 src/ 下, 入口文件为 src/index.js

组件开发时搭建一个本地测试用的场景(包括测试用的模型及其数据), 使用 vite 实时编译和热更新, 方便调试和开发, 测试用的场景不发布, 只作为开发时的工具

组件发布时使用 vite 打包成 esm、umd 格式, 以便在各种环境中使用, 包括浏览器直接引入和 npm 包管理器安装使用

vite 使用最新的 8 版本, 使用 rolldown 进行打包

暂不考虑复杂边界场景, 如异常模型命名自动修复、多层贴图混合、超大模型性能自适配等, 这些可以在后续版本中逐步完善

## 兼容性

使用组件时最低兼容 Node.js 16(不是硬性要求, 但是需要在 README 中写明如果使用的 Node.js 版本太低需要做什么, 开发组件时不必兼容低版本 Node.js), 不支持 IE 浏览器, 只支持现代浏览器
